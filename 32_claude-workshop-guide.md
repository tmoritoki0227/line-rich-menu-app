> **このドキュメントの目的**

> Claudeの特性を活かすことでより効率よく進められます。
> Claude ProまたはClaude Teamプランの契約が必要です。

## 作るもの・システム構成・作業ステップ

31番（`31_ai-workshop-guide.md`）と同じです。このドキュメントは「Claudeの使い方」に特化した補足です。

---

## Claudeを選ぶ理由

| 特徴 | 説明 |
|---|---|
| **長い文脈を保持できる** | 会話が長くなっても前の内容を覚えている。途中で状況説明をし直す手間が少ない |
| **コードと説明を一緒に出せる** | コードだけでなく「なぜそう書くか」も説明してくれる |
| **ファイルを読める** | エラーログやコードファイルを貼ると内容を理解して回答する |
| **修正指示に強い** | 「この部分だけ変えて」「もっとシンプルにして」が通じやすい |

## Claudeとの進め方

### セッションの始め方

最初に今回のプロジェクト全体を宣言します。これをすることでClaude が文脈を保持したまま会話が続けられます。

```
これからLINEとAWSを使った家計簿ボットを作ります。
使う技術はNode.js、AWS（Lambda/DynamoDB/S3/CloudFront/APIGateway）、
LINEのLIFFとMessaging APIです。
完成形はLINEのリッチメニューからWebアプリを開いて
支出の登録と月次集計ができるものです。
順番に作業を進めていくので手伝ってください。
```

### ステップごとの進め方

各ステップの作業をひとつずつClaude に依頼します。一度に全部頼まず、1ステップずつ確認しながら進めるのがコツです。

**例：DynamoDBのテーブル設計を頼む**
```
ステップ3のDynamoDB設計をしたいです。
保存したいデータは以下です。
- ユーザーID（LINEのuserID）
- 日付
- 金額
- 品物名

このデータに適したDynamoDBのテーブル設計を提案してください。
パーティションキーとソートキーの選び方も説明してください。
```

**例：エラーを解決してもらう**
```
以下のエラーが出ています。

[エラーメッセージをそのまま貼る]

現在のコードはこれです。

[コードをそのまま貼る]

原因と修正方法を教えてください。
```

### Claudeへの効果的な指示パターン

**コードを生成してもらう**
```
〇〇する処理をNode.jsで書いてください。
条件は以下です。
- 〇〇
- 〇〇
コメントも日本語でつけてください。
```

**概念を理解する**
```
〇〇とは何ですか？
初心者向けに、具体例を使って説明してください。
```

**選択肢を比較する**
```
〇〇を実現するために〇〇と〇〇という方法があります。
それぞれのメリット・デメリットを比較して、
今回のケース（〇〇）にはどちらが向いているか教えてください。
```

**設計をレビューしてもらう**
```
以下の設計を考えました。問題点や改善点があれば教えてください。
[設計内容を書く]
```

## Claudeを使うときの注意点

**同じ会話を続ける**

新しいチャットを始めると前の内容をClaude は覚えていません。同じ作業は同じ会話の中で続けてください。

**「なぜ」も聞く**

コードをもらうだけでなく、なぜそう書くのかを聞くと理解が深まります。
```
このコードはなぜこういう書き方をしているのですか？
```

**うまくいかないときは状況を詳しく伝える**

「動きません」だけでなく、何をしたときにどうなったかを具体的に伝えます。
```
△△の手順で操作したところ、
××という画面になり、〇〇というエラーが出ました。
期待していた動作は□□です。
```

**Claudeの回答が間違っていることもある**

Claudeは誤った情報を自信ありげに答えることがあります。特にAWSの細かい設定やLINEのAPIの仕様については、公式ドキュメントと照合する習慣をつけてください。

---

## Claudeに聞くと特に効果的な場面

| 場面 | 理由 |
|---|---|
| エラーメッセージの解読 | エラーの意味と原因を日本語で説明してくれる |
| IAMポリシーの設定 | 権限設定は複雑なのでClaude に生成させると速い |
| コードのリファクタリング | 動いているコードをきれいにしてもらえる |
| 概念の理解（非同期処理など） | 図や例を使って丁寧に説明してくれる |
| デバッグ | コードとエラーを貼るだけで原因を特定してくれることが多い |
